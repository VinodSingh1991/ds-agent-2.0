================================================================================
SUPPORTED OBJECT TYPES - Quick Reference
================================================================================

The Disposable UI Agent now supports 7+ object types with realistic mock data.
All patterns work across all object types automatically!

================================================================================
1. LEAD
================================================================================
Fields:
  - id (number)
  - name (string) - Company name
  - revenue (currency)
  - status (enum) - qualified, negotiation, won
  - contact (string) - Contact person
  - created (date)

Example Query:
  "show me all leads"
  "display top 5 leads by revenue"
  "show qualified leads"

Best Patterns:
  - user_avatar_list (if has contact photos)
  - data_table_pattern (for comparison)
  - metric_dashboard (for KPIs)

================================================================================
2. CONTACT / USER
================================================================================

  - id (number)
  - name (string)
  - email (email)

  - phone (phone)

================================================================================

================================================================================
PATTERN SELECTION LOGIC
================================================================================

The agent automatically selects the best pattern based on:

1. Query Intent:
   - "list", "show all" → List patterns
   - "details", "view" → Detail patterns
   - "dashboard", "metrics" → Dashboard patterns

2. Data Characteristics:
   - Has images/photos → user_avatar_list, card_grid_pattern
   - Has status field → status_list_compact
   - Many numeric fields → data_table_pattern, metric_dashboard
   - Single record → detail_view_card

3. Object Type Hints:
   - Contact/User → user_avatar_list
   - Product → card_grid_pattern
   - Case/Ticket → status_list_compact
   - Account/Branch → data_table_pattern

================================================================================
ADDING NEW OBJECT TYPES
================================================================================

To add a new object type:

1. Add mock data in: agent/data_fetcher.py
2. Follow the pattern:
   elif "your_object" in object_type:
       return [
           {"id": 1, "field1": "value1", ...},
           ...
       ]

3. No need to modify patterns - they work automatically!
